PRUEBAS GUI - INGRESO MANUAL DE DATOS (SIN SQL)

Objetivo
Registrar clientes y prestamos desde la interfaz grafica para validar resultados distintos de evaluacion:
1) Aprobado
2) Contrapropuesta
3) Rechazado
4) Revision_manual

Regla principal
- NO usar phpMyAdmin para insertar datos.
- TODO se captura desde pantallas del sistema.


============================================================
1. RUTA DE PRUEBA (100% GUI)
============================================================

A. Modulo Clientes:
- Registrar 4 clientes con los datos de este documento.

B. Modulo Prestamos:
- Buscar cada cliente.
- Crear un prestamo personal por cliente.
- Enviar con boton "Evaluar prestamo".

C. Pantalla Evaluacion:
- El sistema redirige automaticamente a Evaluacion.
- Esperar fin de evaluacion y abrir resultados.

D. Verificacion:
- Revisar decision final en Resultado o Contrapropuesta.
- Opcional: revisar el mismo prestamo en Seguimiento.


============================================================
2. CAMPOS IMPORTANTES QUE DEBES LLENAR
============================================================

En Clientes (Registrar cliente), son claves para la evaluacion:
- Identidad: nombre, apellido, fecha nacimiento, tipo/numero documento, genero, telefono, email.
- Socioeconomico: tipo vivienda, dependientes, direccion.
- Laboral: ocupacion, empresa, antiguedad laboral, sector laboral, ingresos mensuales, fuente ingresos, egresos mensuales.

Validaciones a respetar:
- Cliente mayor de edad.
- Ingresos mensuales >= 10000.
- Ingresos > egresos.

En Prestamos (Solicitud de prestamo personal):
- Monto solicitado.
- Plazo (meses).
- Gastos mensuales (pestana Datos financieros).
- Politica de cancelacion.
- Luego: boton "Evaluar prestamo".

============================================================
4. COMO EJECUTAR CADA PRUEBA (PASOS CORTOS)
============================================================

Para cada caso:
1. Ir a Clientes -> Registrar cliente.
2. Llenar Identidad, Socioeconomico y Laboral.
3. Guardar cliente.
4. Ir a Prestamos.
5. Buscar cliente por nombre o documento.
6. Click en Prestamo personal.
7. Completar monto, plazo y gastos mensuales.
8. Click en Evaluar prestamo.
9. Esperar pantalla de Evaluacion.
10. Click en Ver resultados completos (o Ver contrapropuestas).


============================================================
5. SI NO COINCIDE EXACTAMENTE EL RESULTADO
============================================================

En algunos ambientes pueden variar reglas activas, score externo o intervalos.
Si pasa eso, ajusta solo estos campos y reintenta:

Para mover a Aprobado:
- Bajar monto solicitado.
- Aumentar plazo.
- Bajar gastos mensuales.

Para mover a Contrapropuesta:
- Subir monto hasta que la cuota supere ligeramente la capacidad.
- Mantener perfil de riesgo medio (no extremo).

Para mover a Rechazado:
- Subir monto y/o bajar plazo para elevar cuota.
- Subir gastos mensuales para reducir capacidad.

Para mover a Revision_manual:
- Mantener puntaje medio y agregar una friccion moderada (ej. gasto algo alto, pero no extremo).
